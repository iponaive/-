# 專案名稱：Word 批次轉 PDF 暨自動化頁面處理工具 

## 專案簡介
本工具專為處理大量 Word 參考文件設計，利用 PowerShell 驅動 Word COM 自動化介面，實現高效且標準化的 PDF 轉檔流程。程式除了執行格式轉換外，更包含文件清理邏輯，確保輸出文件無冗餘修訂資訊並具備統一的視覺識別。

## 核心功能與技術邏輯
* **文件安全性清理**：
    * **自動刪除註解**：一鍵清空文件中所有隱藏註解（Comments），防止敏感資訊外流。
    * **隱藏修訂追蹤**：自動關閉修訂顯示，確保 PDF 僅呈現最終定案版本。
* **首頁印章精確定位**：
    * 採用「相對於頁面 (Page-relative)」的絕對座標定位技術。
    * 自動將 **PBC 印章** 放置於首頁左上角（Top: 10, Left: 20），避開正文內容。
* **智慧檔案解析與動態頁尾**：
    * **標題解析**：自動擷取檔名中底線（_）前的文字作為識別標題。
    * **專業頁尾**：
        * 左側：署名 `By Ariel Lin`。
        * 右側：顯示 `[標題] P.[自動分頁碼]`。
        * 設定：統一 Times New Roman 字體與 Size 6 規格。

## 邏輯範例
* **原始檔案**：`Contract_2026_Draft.docx` (含內部註解與修訂軌跡)
* **處理程序**：
    1. 執行 `Comments.DeleteAll()` 清除所有註解。
    2. 擷取 `Contract` 作為文件識別碼。
    3. 於首頁左上角植入 PBC 識別印章。
* **輸出結果**：乾淨、標準化的 `Contract_2026_Draft.pdf`。

## 技術實作
* **開發語言**：PowerShell
* **核心技術點**：
    * **COM 自動化**：驅動 `Word.Application` 進行背景處理。
    * **頁面佈局控制**：精準計算 `$cmToPoints` (28.35) 進行圖檔縮放與定位。
    * **多節頁尾處理**：遍歷 `Footers.Item(1, 2)` 確保不同頁面設定下均能正確顯示。

## 執行流程
1. 確保 `pbc_stamp.png` 圖檔與腳本位於同一目錄。
2. 將待處理的 Word 文件放入該路徑。
3. 執行 PowerShell 腳本。
4. 於同路徑取得清理完成且帶有印章與頁尾的 PDF。
